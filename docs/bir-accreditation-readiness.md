# BIR Accreditation Readiness - System Checklist

Last updated: March 19, 2026 (Asia/Manila)

## Implemented in System

- Live EIS submission mode now supports strict `REMOTE_REQUIRED` when endpoint is not configured.
- EIS summary now exposes:
  - `mode`
  - `endpointConfigured`
  - `liveSubmissionReady`
  - `readinessIssues`
- POS payment flow now posts sale directly to `/api/pos/transactions` for server-issued OR/reference at checkout.
- POS can enforce live issuance (`REQUIRE_LIVE_RECEIPT_ISSUANCE=true` by default).
- Receipt actions are functional on Windows:
  - Print
  - Reprint
  - Email draft handoff
- Receipt action events are now synced as `receipt_events` and written into the backend audit trail as:
  - `RECEIPT_PRINT`
  - `RECEIPT_REPRINT`
  - `RECEIPT_EMAIL`

## Runtime Flags

- `EIS_ALLOW_SIMULATION=false` (recommended for accreditation readiness)
- `EIS_ENDPOINT_URL=<live endpoint>`
- `EIS_API_KEY=<issued key>`
- `EIS_TAXPAYER_TIN=<registered TIN>`
- `EIS_SIGNING_SECRET=<non-default secret>`
- `REQUIRE_LIVE_RECEIPT_ISSUANCE=true` (POS app)
- `ALLOW_OFFLINE_SALE_QUEUE=false` (POS app)
