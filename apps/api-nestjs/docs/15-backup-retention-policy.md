# BIGTIME POS — Database Backup and Ten-Year Retention Policy

**Document No.:** 15  
**Annex B Reference:** Item — Database backup preserved for mandatory ten-year retention period  
**Date Prepared:** 2026-04-16

---

## 1. Purpose

This document describes the backup schedule, storage procedures, and ten-year retention commitment for all transaction data generated by BIGTIME POS, in compliance with BIR requirements under RMO No. 24-2023 and related revenue regulations.

---

## 2. Data Covered

The following data must be backed up and retained:

| Data Category | Storage Location | Retention |
|---|---|---|
| Sales transactions (SI/OR records) | PostgreSQL — central API database | 10 years |
| Void and refund records | PostgreSQL — central API database | 10 years |
| Z-readings and X-readings | PostgreSQL — central API database | 10 years |
| Audit trail (hash-chained log) | PostgreSQL — `audit_log` table | 10 years |
| BIR E-Sales submissions | PostgreSQL — EIS submission queue | 10 years |
| POS offline queue (local) | Flutter app — Drift SQLite database | Until sync confirmed |
| Generated CSV/DAT reports | File storage (server or cloud bucket) | 10 years |
| Employee and shift records | PostgreSQL | 10 years |

---

## 3. Backup Schedule

| Backup Type | Frequency | Method |
|---|---|---|
| Full database dump | Daily (end of business day) | `pg_dump` to encrypted archive |
| Incremental backup | Every 4 hours during operating hours | WAL archiving (PostgreSQL streaming replication) |
| Offsite/cloud copy | Daily | Transfer to cloud storage (e.g., AWS S3, Google Cloud Storage, or equivalent) |
| Report archive | Weekly | Export all generated CSVs to long-term archive storage |

---

## 4. Backup Retention Schedule

| Period | Storage Tier |
|---|---|
| Last 30 days | Hot storage (fast restore) |
| 30 days – 1 year | Warm storage (standard cloud) |
| 1 year – 10 years | Cold/archival storage (low-cost, compliance-grade) |

All backups must be retained for a minimum of **ten (10) years** from the date of the transaction, as required by BIR regulations.

---

## 5. Backup Integrity

- All backups are verified via checksum after creation.
- Monthly restore tests are conducted to confirm backup integrity.
- Backup logs are maintained and available for BIR inspection.

---

## 6. Responsibility

| Role | Responsibility |
|---|---|
| System Administrator | Execute and monitor daily backup jobs |
| IT Manager / Owner | Review monthly backup reports and test restores |
| BIR Compliance Officer | Confirm retention compliance annually |

---

## 7. Disaster Recovery

In the event of data loss, the restoration procedure is:

1. Identify the most recent verified backup
2. Restore PostgreSQL from `pg_dump` archive
3. Apply WAL incremental logs to minimize data loss
4. Verify transaction integrity via audit trail hash chain
5. Notify BIR if any transaction records cannot be recovered

---

## 8. Declaration

The deploying organization confirms that this backup policy is implemented in the production environment and that transaction data is preserved for the mandatory ten-year retention period as required by BIR regulations.
